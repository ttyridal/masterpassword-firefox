MasterPassword Release 2.9.0-beta

Thank you for using MasterPassword for browsers! The easiest way to handle site-unique password on the web. If you want to understand what this is all about, please see here.

You've chosen to install a beta release of this addon. Thank you for beeing an early adopter! If you find any bugs or missing features, please report them on the issue tracker.

If you have any issues with with addon, please report them on the issue tracker.

Users of OS' key-vault

If you're using GNOME Keyring, KWallet, OSX Keychain or Windows Password Vault to store your master key, you'll need to download a separate helper app. Please refer to the wiki for further information.

New data format

This version of MasterPassword changes the internal representation for sites. This is done for a couple of reasons. First, the export file can only represent a sitename once. The addon could previously have a sitename with different configurations registered on different websites.

Example:
You use your google account with the site-name set to google.com and generation set to 2. You configured this when visiting gmail.com (the url you where at when opening masterpassword).
At a later time, you visit youtube.com and you want to login with your google account.
Previously you would have to enter the settings again. google.com at gmail.com and youtube.com was considered two unrelated things. This is no longer the case. There can only be one google.com site-name, and when you change the site-name to google.com (when visiting youtube) masterpassword will give you that the correct setting, with the generation correctly set to 2. And, if you update the generation to 3, it will change in all places where you use google.com as a site-name.
You can observe in the configuration/stored-sites, that google.com will be matched to both youtube.com and gmail.com.
You can of course still have multiple google accounts, user1@google.com, user2@google.com etc.

Furthermore, the new representation is better suited for working around storage limitations imposed by the browser.

Please head over to the configuration view to execute the upgrade.
