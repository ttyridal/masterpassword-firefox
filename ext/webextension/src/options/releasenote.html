MasterPassword Release 3.0

Thank you for using MasterPassword for browsers! The easiest way to handle site-unique password on the web. If you want to understand what this is all about, please see here.

You've chosen to install a beta release of this addon. Thank you for beeing an early adopter! If you find any bugs or missing features, please report them on the issue tracker.

If you have any issues with with addon, please report them on the issue tracker.

Users of OS' key-vault

If you're using GNOME Keyring, KWallet, OSX Keychain or Windows Password Vault to store your master key, you'll need to download a separate helper app. Please refer to the wiki for further information.

Changes since v2.3.1 (firefox) / 2.1.5 (chrome)

  • Deny multiple instances of site with different settings (see details below)
  • Added import/export of mpjson format (can now store url match info)
  • Added import mpsites/mpjson button, in addition to drag-n-drop
  • Reduced set of required permissions
  • Merged Firefox and Chrome codebases
  • Fix for many second level domains (edu.nz etc) not beeing recognized and giving wrong default sitename
  • Fix extension not working for sites opened as a webapp ("create application shortcut") (thanks to Jesse Morgan)
  • Fix for frameworks that did not detect password inserted (thanks to Andrew Armenia)

New data format

This version of MasterPassword changes the internal representation for sites. This is done for a couple of reasons. First, the export file can only represent a sitename once. The addon could previously have a sitename with different configurations registered on different websites (ie. sitename google.com could have different generation number when visiting google.co.uk and google.com).

Furthermore, the new representation is better suited for working around storage limitations imposed by the browser.

Please head over to the configuration view to execute the upgrade.
